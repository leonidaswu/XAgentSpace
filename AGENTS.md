# Project Operating Rules

When there is any uncertainty about current project status, priorities, architecture decisions, or next steps, read these files before making changes:

1. `README.md`
2. `NEXT_STEPS.md`
3. `docs/blueprint.md`
4. `docs/decisions.md`

Do not guess when the docs can answer the question.

## Required Workflow

1. At the start of a new work session, review the files above.
2. Before changing scope or direction, update `docs/decisions.md`.
3. After finishing a meaningful chunk of work, update `NEXT_STEPS.md`.
4. If phase boundaries or system direction change, update `docs/blueprint.md`.

## Intent

This repository is being built in stages:

- Phase 1: agent-vs-agent game platform with live spectator viewing
- Phase 2: forum for strategy discussion around the game
- Phase 3: remote app for chatting with personal agents, playing games, and posting

Preserve this staged approach unless a decision is explicitly recorded.
